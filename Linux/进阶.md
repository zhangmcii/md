## find
find 路径 条件 -exec 命令 {} \;
find 路径 条件 -exec 命令 {} +

解释：
-exec	告诉 find 要执行命令
命令	要执行的程序
{}	当前找到的文件
\;	命令结束符
+   命令结束符

\;（旧方式），每个文件执行一次， 会产生很多进程。
而 +（更高效），一次传多个文件，性能更好。

# awk : 文本分析工具 (可以逐行读取文本文件，并提供类似编程语言的功能)
语法：
```
awk 'pattern { action }' file
```

如果只写：awk 'pattern'， 其实等价：awk 'pattern { print }'

10.打印文件faces.txt，但只打印每个重复行的第一个实例，即使重复行之间没有相邻出现。
请注意，顺序很重要，因此在去除重复项之前不要对行进行排序。
```
awk '!seen[$0]++' faces.txt
```

\$0 ： 当前整行
\$1 ： 第一列
\$2 ： 第二列
NR ： 行号
NF ： 字段数量
seen[$0]:  是数组


awk 支持 关联数组（像 Python dict）,
```
seen["hello"] = 1
seen["world"] = 1
```
所以seen[$0]意思是：用这一行当 key

seen[$0]++ 的行为是先返回旧值，再加 1。



11.文件random-numbers.txt包含一个由100个随机整数组成的列表。请打印该文件中包含的唯一素数的数量。
```
awk '
function prime(n,i){
 if(n<2) return 0
 for(i=2;i*i<=n;i++)
  if(n%i==0) return 0
 return 1
}
prime($1){print $1}
' random-numbers.txt | sort -u | wc -l
```
逻辑：
awk      → 找素数
sort -u  → 去重
wc -l    → 统计


function prime(n,i) 形参是2个，为什么prime($1)只传1个？
少传的参数会自动变成 空值（0 或 ""）。

其实这里的i传不传都不会被用上，因为进入函数i自动被初始化为2。那么这样定义不就行了function prime(n)？
可以，这里的 i 不是局部变量，它会变成 全局变量。如果别的代码也用了 i，就可能被污染。
awk 规定：function f(arg1,arg2,   local1,local2)，参数后面那些没传值的变量会变成局部变量。

<details>
<summary>prime(n,i)逻辑为什么能判断素数?</summary>
素数定义: 只能被1和自己整除的数

for (i = 2; i * i <= n; i++)： 从 2 开始试除，一直试到 √n
为什么只需要检查到 √n？
如果 n 不是素数：
```
n = a x b
```
假设：a ≤ b
那么必然有：a ≤ √n
否则: a > √n, b > √n
就会变成：a × b > n 矛盾
所以：任何合数，必然存在 ≤ √n 的因子
因此只需要检查：2 → √n 就够了

所以几乎所有简单素数判断都是这个套路。

</details>
